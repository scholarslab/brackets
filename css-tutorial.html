Your Guide to CSS

CSS stands for "Cascading Style Sheet". Coding with CSS will help style the content in your HTML document. CSS describes how the elements in your HTML document should be displayed and helps format the layout. Using this style language you will select or reference certain things, or elements, on a webpage in order to change and define properties such as typography, color, and sizing. Think of your HTML document as laying out the bones and body of your webpage while CSS gives it its distinct appearance and look.

Before you dive into the css, make sure you have some idea of the design aesthetic that you would like to convey on your website through the typology, color and layout of your webpage. Typology will help you create and convey meaning through the choices you make about the font, size, color, layout and alignment of your website. Consider the following:

  1. Look through websites or personal portfolio pages that resonate with you aesthetically.
  2. Our Design Ethos and Brainstorming page will help guide you though some of these considerations and questions.

In this example there are two selectors: html { } and h1{ }. While the properties in html { } apply to the entirety of your HTML content, you can target specific sections of your webpage using a variety of selectors. h1{ } is one of these selectors. In this code block you've been invited to defined a different color for the h1 heading. Because the h1 element is an ancestor or descendant (i.e. it comes after) of the root element, the CSS code and will use this last definition to encode the h1 element of your HTML content. This means that the properties we have defined for h1 will override those specified within the html selector. In other words, the new font color property you define for the level one headings overrides the font color properites definied in the root HTML element, so instead of grey the h1 headings appears in the color of your choice.

The reason this happens is because CSS is read from top to bottom and whatever the last definition it encounters is the way it will encode your webpage. This allows you to lay out a hierarchy of properties and values. This principle or rule is called "cascade". Does that sound familiar? Remember what CSS stands for? I'll give you a moment. Yep, CSS is an acronym for Cascading Style Sheet and this is precisely what is meant by cascading.

One thing to note is that properties you define under a particular descendant selector, such h1, will apply to any HTML content that you have tagged in that way. This means that all the HTML content tagged as h1 on your webpage will appear in the color or font-family you have defined under that selector in your CSS.

In order to define properties for one single unique element in your HTML you need to use and ID. ID is a unique identifier for your page content that allows you to code css to one single specific section or word.
